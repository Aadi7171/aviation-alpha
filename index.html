Alpha Sources

Three signals,
one composite score

Each signal targets a distinct dimension of airline stress. Together they form a robust M&A pressure indicator.

SIGNAL 01
✈
Route Overlap

Jaccard similarity measures competitive overlap between carrier route networks. High overlap historically precedes merger activity — e.g. AA/US Airways consolidation in 2013.

Jaccard(A,B) = |routes_A ∩ routes_B|
              / |routes_A ∪ routes_B|
// composite weight: 0.40
SIGNAL 02
⚡
Disruption Spikes

Rolling z-score on cancellation and delay rates. A 2.5σ spike flags abnormal operational stress — a leading indicator of financial distress for a carrier.

z = (cancel_rate − μ) / σ
spike = |z| > 2.5
// composite weight: 0.25
SIGNAL 03
🔗
Network Vulnerability

NetworkX graph metrics — betweenness centrality and clustering coefficients — reveal structural fragility. Low clustering = no redundant routes = M&A target profile.

betweenness_centrality(G)
clustering_coefficient(G)
// composite weight: 0.35

Methodology

Composite M&A
pressure score

The three signals are combined into a single composite score reflecting the overall acquisition likelihood for any carrier in the dataset.

score = 0.40 × max_route_overlap
      + 0.35 × network_vulnerability
      + 0.25 × disruption_spike_rate
  • 01
    Data IngestionBTS T-100 segment data and OpenFlights routes. FlightAware AeroAPI v4 for real-time signals.
  • 02
    Feature EngineeringJaccard similarity, rolling z-scores, and NetworkX graph metrics computed per carrier.
  • 03
    Econometric ValidationGranger causality tests confirm predictive power at p<0.05.
  • 04
    BacktestingSignal converted to long/short P&L. Evaluated vs SPY and XAL airline ETF.

Quick Start

Get running in 4 steps

STEP 01
Clone & Install

Clone the repo and install Python dependencies.

git clone github.com/
Aadi7171/aviation-alpha
cd aviation-alpha
pip install -r requirements.txt
STEP 02
Configure API Key

Optional. Free tier: 500 req/month.

cp .env.example .env
# Edit .env and add:
FLIGHTAWARE_API_KEY=your_key
STEP 03
Launch Dashboard

Start the 6-tab Streamlit dashboard.

streamlit run src/dashboard/app.py

# Opens localhost:8501
STEP 04
Run Tests

Validate the feature pipeline.

python -m pytest tests/ -v

# All tests pass ✓
